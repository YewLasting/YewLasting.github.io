VS2019配置GDAL-3.2.2

GDAL3.0编译要求PROJ投影库，而PROJ又要求SQLite3环境，所以在编译GDAL之前我们需要先编译好SQLite和PROJ.
GDAL本身没有自带HDF库，如果需要读取HDF4/HDF5数据的话还需要集成GEOS库和HDF4/HDF5库。

系统环境：Win10 专业版 20H2

VS版本：Visual Studio 2019

GDAL版本：GDAL 3.2.2

CMake版本：CMake 3.19.7

SQLite版本：SQLite 3.35.2

PROJ版本：PROJ 6.3.2 不建议换为其他版本，7以上的版本在编译GDAL时还需要其他的lib文件

GEOS版本：GEOS 3.7.3 不建议换为其他版本。下载后还需要编译

HDF4版本：HDF4 4.2.6 不建议换为其他版本。下载的就是已经编译好的

HDF5版本：HDF5 1.8.7 不建议换为其他版本。下载的就是已经编译好的

以上安装包百度云链接，提取码：ageg

三.编译安装GDAL

  1. 下载地址：GDAL

  2. 此处采取的是动态编译GDAL库(静态编译、动态编译、静态链接库和动态链接库理解)

  3. 首先在GDAL文件夹中创建一个GDALPackages322文件夹，将压缩包中的文件解压到里面：

  4. 接下来在开始编译GDAL之前，首先简单了解一下nmake.opt文件中变量的意义(链接)：

    项目 Value
    MSVC_VER 编译器版本
    WIN64 是否编译64位版本
    GDAL_HOME 生成文件的目录
    DLLBUILD 是否动态编译，1为是，0为否

  5. 进入GDALPackages322文件夹中，找到nmake.opt文件，用VS2019或者VSCode打开。

  6. 修改GDAL源码中的nmake.opt文件（需要修改5处）

    • 42行左右的MSVC_VER=设置为1928，具体数字根据自己的MSVC版本设置，参考这个；

    • 66行左右的GDAL_HOME =设置成想要的位置，我在GDAL文件夹下新创建一个GDALHome322文件夹后，将此处设置成E:\GDAL\GDALHome322；

    • 213行左右的WIN64=YES，删掉前面的#取消注释；

    • 260行左右的PROJ_INCLUDE设置为第二步编译生成的PROJ时的include文件夹：E:\GDAL\OSGeo4W\include，其中PROJ_INCLUDE中的-I后为地址。

      和264行左右的PROJ_LIBRARY设置成第二步编译生成的PROJ时的lib文件夹：E:\GDAL\OSGeo4W\lib\proj.lib，记得把.lib的名称改对。

    • 650行左右的SQLITE_INC设置为第一步编译生成的SQLite时的include文件夹：E:\GDAL\SQLite\include，其中SQLITE_INC中的-I后为地址。

    • 651行左右的SQLITE_LIB设置设置为第一步编译生成的SQLite时的lib文件夹：E:\GDAL\SQLite\lib\SQLite3.lib，记得将lib文件的名称修改正确。

    • 至此设置完成，保存文件。

  7. 管理员运行x64 Native Tools Command Prompt for VS 2019（在开始菜单中），先输入E:，进入磁盘E，然后cd E:\GDAL\GDALPackages322进入GDAL文件夹下：然后依次输入以下3行命令：

    nmake /f makefile.vc
nmake /f makefile.vc install
nmake /f makefile.vc devinstall

    完成后会在上面设置的E:\GDAL\GDALHome322文件夹中生成bin、lib、data、include四个文件夹

  8. 最后在VS2019中设置库的引用，参考博客中的6-8步（第8步是测试）。
    其中，第6步 属性管理器的配置中,VS2019中没有Microsoft.Cpp.x64.user文件，可以把VS2017中带的MSBuild文件夹整体复制到路径C:\Users\dell\AppData\Local\Microsoft下，再重新打开属性管理器继续配置就可以看到Microsoft.Cpp.x64.user了。
    在此附上MSBuild文件链接，提取码8ouy（感谢前辈的分享）。

    在GDALHome322文件夹中新建立一个supportlibs文件夹，将geos-3.7.3.tar.bz2中的内容解压到里面：

  1. 修改GEOS源码文件夹中的nmake.opt文件：

  2. 56行的WIN64=设置成YES；

    166和167行的GEOS_MSVC和GEOS_MSC 分别按照自己的MSVC版本修改，参考这个；

    autogen.bat

    完成编译。生成的lib和dll文件在src文件夹下，共五个。

五.集成GEOS+HDF4+HDF5

  1. 下载HDF4 4.2.6 和HDF5 1.8.7 ，并解压到GDALHome中的supportlibs文件夹中：

  1. 再次修改GDAL源码中的nmake.opt文件。

    GEOS处（找到#Uncomment for GEOS support (GEOS >= 3.1.0 required)）
    将以下代码

GEOS_DIR=D:\WORK\GDALwarmerda\supportlibs\geos-3.7.3
GEOS_CFLAGS = -I$(GEOS_DIR)/capi -I$(GEOS_DIR)/include -DHAVE_GEOS
GEOS_LIB     = $(GEOS_DIR)/src/geos_c_i.lib

修改为：

GEOS_DIR=E:\GDAL\GDALHome322\supportlibs\geos-3.7.3
GEOS_CFLAGS = -I$(GEOS_DIR)/capi -I$(GEOS_DIR)/include -DHAVE_GEOS
GEOS_LIB     = $(GEOS_DIR)/src/geos_c_i.lib

  1. HDF4处将以下代码

    #HDF4_PLUGIN = NO
#HDF4_DIR = D:\warmerda\HDF41r5
#HDF4_LIB = /LIBPATH:$(HDF4_DIR)\lib Ws2_32.lib
#HDF4_INCLUDE = $(HDF4_DIR)\include

    修改为：

    HDF4_PLUGIN = NO
HDF4_DIR = E:\GDAL\GDALHome322\supportlibs\HDF4.2.6_win_x64
HDF4_LIB = $(HDF4_DIR)\dll\hd426m.lib $(HDF4_DIR)\dll\hm426m.lib \
$(HDF4_DIR)\lib\hd426.lib $(HDF4_DIR)\lib\hm426.lib Ws2_32.lib
HDF4_INCLUDE = $(HDF4_DIR)\include

  2. HDF5处（找到# Uncomment the following and update to enable NCSA HDF Release 4 support.）
    分别将以下代码

    #HDF5_PLUGIN = NO
#HDF5_DIR =    c:\warmerda\supportlibs\hdf5\5-164-win
#HDF5_LIB =    $(HDF5_DIR)\dll\hdf5dll.lib


    修改为：

    HDF5_PLUGIN = NO
HDF5_DIR = E:\GDAL\GDALHome322\supportlibs\HDF5-1.8.7_win_x64
HDF5_LIB = $(HDF5_DIR)\dll\hdf5dll.lib

修改完毕，保存文件。

最后打开x64 Native Tools Command Prompt for VS 2019，进入E:\GDAL\GDALPackages322文件夹下，重新运行以下三行命令：

nmake /f makefile.vc
nmake /f makefile.vc install
nmake /f makefile.vc devinstall

完成GDAL编译。

  1. 最后，记得将E:\GDAL\GDALHome322\supportlibs\HDF4.2.6_win_x64\dll文件夹和E:\GDAL\GDALHome322\supportlibs\HDF5-1.8.7_win_x64\dll文件夹中所有的.dll文件拷贝到GDAL编译后存放的目录下的bin文件夹中，也就是E:\GDAL\GDALHome322\bin文件夹中，该文件夹中有gdal302.dll。

    E:\GDAL\GDALHome322\supportlibs\geos-3.7.3\src中的geos_c.dll文件也拷贝到E:\GDAL\GDALHome322\bin目录中。

    否则会提示你找不到xxx.dll文件。
